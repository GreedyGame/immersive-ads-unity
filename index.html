Native Ad Integration

SDK Version - 1.2.1 Last updated @26/05/2023

Before Integration

  1. Make sure you are using Unity version 2019 or above
  1. Android platform is selected for your Unity project
  1. You have the Ad Mob app id ready with you.

Editor Integration Steps

  1. Download and import the latest version of the Google Mobile Ads plugin.
  1. Download and import the latest version of the PubScale Native ads plugin.
  1. Force resolve the dependencies from Assets/External Dependency Manager/Android Resolver/Force Resolve if the resolver did not open automatically.
  1. Enter your Ad Mob app id inside the editor window pop up
    1. Go to Assets/Google Mobile Ads/Settings

    b. In the Settings Window add your Android application id.

Scene Setup

  1. Drag and drop the Native ad manager prefab from Assets/PubscaleSDK/NativeAds/Prefabs/NativeAdManager.prefab into your gameplay scene.
  1. Drag and drop any of the Native ad unit prefab from Assets/PubscaleSDK/NativeAds/Prefabs/AdType into your gameplay scene. (Note: If using the “NativeAd_ScreenSpaceCamera” prefab then put it under any Canvas in Screen Space-Camera render mode in order to see ads on it.)
  1. Put your native ad unit id inside the Native Ad Holder script. Leave the default test id if you don’t have a native ad unit id.
  1. To test the integration always use test mode inside the Native Ad Manager prefab. (This will enable test ads and shows a debug UI over the native ad prefab for Impression and click check)
  1. Hit the play button in the editor to ensure PubScale dummy test ads are visible and there are no errors and warnings. Also please note that the impression and click don’t work inside Editor.
  1. Finally To Test Impressions and Clicks, build the application to the actual device and check you must be able to see the impression indicator turning green when the ad comes inside the camera view and the click indicator turning green when you tap on the ad.

Your Integration is now complete! 🎉

Debugging

Ad Loading Failed

  1. Make sure you have added the app id inside the google ad mob settings window.
  1. In the case of live ad id (Test mode off), this may happen if there is no fill from the Google server.

No Impression but Click

  1. If it’s a 3d game the camera and the ad unit must be on the side + Z world axis in order to get an impression.
  1. Make sure there is no collider between the main camera and the ad Unit.
  1. If you are instantiating ad prefabs in real-time try disabling the strip engine code option from the build option.

  1. If you have multiple camera’s on the scene make sure the camera which is looking toward the ad unit has the priority depth value.
  1. Make sure the Unity Event system component is present in the scene.

Impression but No Click

  1. Make sure there is no collider between the main camera and the ad Unit.
  1. Make sure the Unity Event system component is present in the scene.
